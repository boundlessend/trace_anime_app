# TraceAnime - PRD

Документ продукта: что реализовано сейчас и что рассматривается дальше.
Раздел 5 - реализованный scope версий 1.1-1.2; раздел 7 - идеи на будущее (не утверждены).

## 1. Обзор

TraceAnime - утилита в menu bar macOS для поиска аниме-сцены по кадру через API [trace.moe](https://trace.moe/).
Пользователь даёт картинку (URL, буфер обмена, drag and drop, файл), приложение возвращает аниме, эпизод, таймкод и превью момента.

- Платформа: macOS 14+, только menu bar (`LSUIElement`).
- Аудитория: те, кто хочет быстро опознать аниме по скриншоту или кадру.
- Распространение: DMG из GitHub Releases, ad-hoc подпись.

## 2. Архитектура и стек

- Swift 5.9 tools, SwiftUI, SwiftPM, единый executable target `TraceAnime`.
- Функциональный стиль: модели - `struct`/`enum`, бизнес-логика - чистые функции, классы только для внешних систем (API-клиент, провайдеры ввода, хранилища).
- Хранение: `UserDefaults` (`AppSettingsStorage`, `LibraryStorage`).
- Сеть: `TraceMoeClient` поверх `URLSession`, таймаут 30s, кэш отключён.
- Локализация: функция `t(...)` (`Localization.swift`), языки EN / RU / FR.
- CI/CD: build (debug+release), swift-format lint, release по тегу `v*` (сборка DMG), Dependabot.

## 3. Реализовано (текущая версия)

### Ввод и поиск
- Поиск по URL изображения.
- Поиск изображения из буфера обмена.
- Drag and drop: файл или изображение.
- Выбор файла с диска.
- Захват области экрана настраиваемым глобальным хоткеем (по умолчанию Option+Cmd+S) с поиском из буфера.
- Опция `cutBorders` (обрезка чёрных полос перед поиском).
- Фильтр по `anilistID`.
- Лимит загрузки 25 MB, превью-картинка до 10 MB.

### Результаты
- Список совпадений: название (romaji / native / english), эпизод, таймкод `from-to`, similarity %.
- Inline-воспроизведение превью-видео момента, mute.
- Размер превью S / M / L.
- Открытие страницы аниме на AniList и MyAnimeList, копирование ссылки и текстовых данных результата.
- Перетаскивание кадра результата наружу (drag) и «Поделиться» через системный share sheet.

### Библиотека
- История поисков с исходным изображением; настраиваемый размер (10 / 20 / 50 / 100), удаление записи и очистка всей истории.
- Избранное (favorites) с исходным изображением, toggle.

### Аккаунт и обслуживание
- Опциональный API-token trace.moe (`x-trace-key`), хранится в настройках.
- Просмотр квоты: id, quota used/total, concurrency, priority.
- Очистка кэша превью.
- Текущая версия в UI и кнопка «Проверить обновления» (проверка GitHub Releases) с окном-результатом и ссылкой на релиз.

### Платформа
- Многоязычный UI EN / RU / FR, переключение на лету.
- Запуск при входе в систему (тумблер в настройках).
- Liquid-glass оформление, перетаскиваемый sticky-копирайт.
- Нативные анимации и тактильные отклики (вращение иконки, `symbolEffect`, `sensoryFeedback`), без внешних зависимостей.

## 4. Ограничения и пробелы

- Нет нотаризации / Developer ID: при первом запуске срабатывает Gatekeeper.
- `SearchQueue` (actor) сейчас просто проксирует клиент - реальной очереди/ограничения конкурентности нет.
- Есть уведомление о новой версии (проверка GitHub Releases), но без авто-установки; полный Sparkle и нотаризация отложены.

## 6. Отложено (бэклог)

- Blur для NSFW-превью (`isAdult` уже в данных).
- Нотаризация релиза (Developer ID) - требует платный Apple Developer аккаунт.
- Полноценный Sparkle с авто-установкой (нужны встраивание фреймворка в bundle, EdDSA-ключи, хостинг appcast).
- Уведомление о завершении долгого поиска при закрытом popover.

## 7. Реализовано в 1.3 и дальнейшие идеи

Приоритет: P1 - заметная ценность; P2 - приятные мелочи. Все нативные, без зависимостей.

| # | Идея | Приоритет | Статус | Заметки |
|---|------|-----------|--------|---------|
| 1 | Настраиваемый хоткей захвата | P1 | Выпущено (1.3) | выбор пресета (⌥⌘S / ⌃⌘S / ⌥⌘A / ⌃⌘A) в настройках, перерегистрация на лету |
| 2 | Поиск по кадру из видеофайла | P1 | Идея | извлечь кадр через AVFoundation, затем обычный поиск; расширяет ввод |
| 3 | Перетаскивание результата наружу | P2 | Выпущено (1.3) | `onDrag` URL кадра в Finder / чат |
| 4 | Поделиться результатом | P2 | Выпущено (1.3) | `ShareLink` (share sheet): ссылка AniList + название и детали |
| 5 | Настройка размера истории | P2 | Выпущено (1.3) | выбор 10 / 20 / 50 / 100 в настройках |
| 6 | Отдельный хоткей «поиск из буфера» | P2 | Идея | без захвата экрана, сразу из буфера обмена |
| 7 | Увеличение кадра / Quick Look | P2 | Идея | клик по превью - крупный просмотр |

Кандидаты из раздела 6 (отложено) тоже остаются в игре: NSFW blur (P1, дёшево), нотаризация (нужен Apple-аккаунт), полный Sparkle, уведомление о завершении поиска.
